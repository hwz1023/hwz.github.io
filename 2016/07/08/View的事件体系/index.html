VelocityTracker

监听控件滑动速度
注意在View的onTouchEvent中获取才有效
@Override
public boolean onTouchEvent(MotionEvent event) {
VelocityTracker velocityTracker = VelocityTracker.obtain();
velocityTracker.addMovement(event);
if (event.getAction() == MotionEvent.ACTION_MOVE) {
//获取速度前需要先计算速度 所需需要先设置计算单位
velocityTracker.computeCurrentVelocity(100);
//计算公式 速度 = (终点位置-起点位置) / 时间段
int xVelocity = (int) velocityTracker.getXVelocity();
int yVelocity = (int) velocityTracker.getYVelocity();
Log.e(“xVelocity”, xVelocity + “”);
Log.e(“yVelocity”, yVelocity + “”);
} else if (event.getAction() == MotionEvent.ACTION_UP) {
//回收内存
velocityTracker.clear();
velocityTracker.recycle();
}

    return super.onTouchEvent(event);
}

    事件分发

    public boolean dispatchTouchEvent(MotionEvent ev)
    用来进行事件的分发，返回结果受到当前View的onTouchEvent和下级View的dispatchTouchEvent方法影响，表示是否消耗事件
    public boolean onInterceptTouchEvent(MotionEvent ev)
    用来拦截事件，如果当前View拦截了某个事件，则在这次事件中该方法不会在调用
    public boolean onTouchEvent(MotionEvent event)
    处理事件，返回值为是否消耗事件
    三者关系
    public boolean dispatchTouchEvent(MotionEvent ev) {
        return super.dispatchTouchEvent(ev);
    boolean consume = false;
    if (onInterceptTouchEvent(ev)) {
        consume = onTouchEvent(ev);
    } else {
        consume = child.dispatchTouchEvent(ev);
    }
    return consume;
}
    onTouchListener 优先级比 onTouchEvent 要高
    结论
  4. 同一事件序列以down事件开始,中间含N个move,以up事件结束
  5. 正常情况一个事件序列只能被一个View拦截
  6. 一个View拦截了事件，那这一个事件序列都只能由他来处理
  7. 某个View处理事件不消耗掉就会将事件重新交给父元素处理
  8. ViewGroup默认不拦截任何事件
  9. View 没有 onInterceptTouchEvent 方法，一旦有点击事件传递给它，那么它的onTouchEvent方法就会被调用
  10. View的onTouchEvent都是默认消耗事件的，除非他是不可点击的（clickable和longclickable同时为false）
  11. View的enable属性(设置View是否可按)不影响onTouchEvent的默认返回值。哪怕一个View是disable状态的只要它的clickable或者longClickable有一个为true那么他的onTouchEvent就返回true
  12. onClick会发生的前提是当前View是可点击的，并且它收到了down和up的事件
  13. 事件传递是由外向内的，事件先传递给父元素，父元素分发给子View通过requestDisallowInterceptTouchEvent
    方法可以在子元素中干预父元素分发过程DOWN除外。

    View的滑动冲突

    内部拦截法
    重写子View的dispatchTouchEvent方法 利用 requestDisallowInterceptTouchEvent处理由父元素还是子元素消耗事件
    @Override
    public boolean dispatchTouchEvent(MotionEvent ev) {
    switch (ev.getAction()) {
        case MotionEvent.ACTION_DOWN:
            getParent().requestDisallowInterceptTouchEvent(true);
            break;
        case MotionEvent.ACTION_MOVE:
            getParent().requestDisallowInterceptTouchEvent(false);
            break;
        case MotionEvent.ACTION_UP:
            getParent().requestDisallowInterceptTouchEvent(true);
            break;
        default:
            break;
    }
    return super.dispatchTouchEvent(ev);
}
    注意

    在父元素onInterceptTouchEvent方法中需要进行如下修改
    不能拦截ACTION_DOWN 否则事件传递不到子View
    @Override

    public boolean onInterceptTouchEvent(MotionEvent ev) {
    int action = ev.getAction();
    if (action == MotionEvent.ACTION_DOWN) {
        return false;
    }
    return true;
}
